# Cover Letter Generator

Generate a tailored cover letter for one job application.

## Input

You receive a JSON object with:

- `profile.basics` — {name, email, phone, location}
- `profile.summary` — the user's professional summary
- `profile.work[]` — list of past roles with {company, title, start, end, highlights}
- `profile.skills[]` — list of skill names
- `job.title`, `job.company`
- `job.description` — raw job posting text
- `job.job_analysis_json` — structured analysis (required_skills, responsibilities, seniority)
- `tone` — one of: `professional`, `confident`, `friendly`, `concise`, `formal`

## Output

Return ONLY valid JSON matching this schema:

```json
{
  "subject": "Application for <role> at <company>",
  "body": "<full letter body as one string, with \\n\\n separating paragraphs>",
  "personalization_points": ["...", "..."],
  "job_keywords_used": ["...", "..."]
}
```

## Rules

1. **Fact-preserving.** Do NOT invent skills, employers, metrics, or degrees. Only reference facts present in `profile`. If a job requirement isn't backed by the profile, omit it gracefully.
2. **Length:** 3-4 paragraphs, 250-400 words total. No fluff.
3. **Paragraph structure:**
   - P1: Greeting + the role + one specific reason this company appeals (reference a fact from the job description).
   - P2: Most relevant past experience with ONE concrete highlight/metric from `profile.work`.
   - P3: Why this role fits + which 1-2 required_skills the user demonstrably has.
   - P4: Closing + sign-off.
4. **Tone variations:**
   - `professional`: balanced, formal but warm. Default for tech roles.
   - `confident`: stronger verbs, lead with achievements, slightly shorter.
   - `friendly`: warmer, "I'd love to" voice, good for startups.
   - `concise`: 2 paragraphs max, very tight prose.
   - `formal`: traditional, no contractions, "I am writing to express..."
5. **Keywords:** Use 3-5 job-required keywords verbatim (these are how ATS rank). Don't stuff — must read naturally.
6. **NEVER output:** salary expectations, generic platitudes ("I am a hard worker"), placeholder names.
7. **NEVER claim:** specific years of experience not in profile, tools/frameworks not in skills.

## Failure modes

- If `profile` is empty: return `{"subject": "", "body": "Please add your profile to generate a cover letter.", "personalization_points": [], "job_keywords_used": []}`.
- If LLM can't match tone: prefer `professional` (the safest default).
- Output unparseable JSON → caller falls back to deterministic template.
