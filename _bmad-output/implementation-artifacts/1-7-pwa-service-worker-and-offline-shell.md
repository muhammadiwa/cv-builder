# Story 1.7: PWA Service Worker & Offline Shell

**Status:** ready-for-dev
**Epic:** 1 — Foundation, Auth & Infrastructure
**Created:** 2026-05-25

---

## User Story

As a mobile user on variable Indonesian internet,
I want the app to install on my home screen and work offline,
So that I can edit my CV even when the connection drops.

---

## Acceptance Criteria

**AC-1:** Given a supported browser, When user visits Lolos, Then PWA install prompt appears after second visit.

**AC-2:** And Service Worker caches app shell (HTML, CSS, JS, fonts) on install.

**AC-3:** And `manifest.json` defines app name, icons (192px, 512px), theme color, and `display: standalone`.

**AC-4:** And offline banner appears within 2 seconds of `navigator.onLine` changing to false.

**AC-5:** And app is functional offline (static pages load, editor opens cached data).

---

## Developer Context

### Technical Requirements

- **Service Worker:** Use `serwist` (Next.js 14 compatible) for SW generation
- **Manifest:** `public/manifest.json` with Lolos branding
- **Offline detection:** `useOnlineStatus` hook using `navigator.onLine` + online/offline events
- **Offline banner:** Yellow bar component: "Kamu sedang offline. Perubahan tersimpan lokal."
- **Cache strategy:** Network-first for API, cache-first for static assets (HTML, CSS, JS, fonts), stale-while-revalidate for images

### Files

- `apps/web/public/manifest.json`
- `apps/web/public/sw.js` (or generated by serwist)
- `apps/web/app/offline/page.tsx` — simple offline fallback page
- `apps/web/components/layout/OfflineBanner.tsx`
- `apps/web/hooks/useOnlineStatus.ts`
- `apps/web/app/layout.tsx` — register SW, wrap with OfflineBanner
