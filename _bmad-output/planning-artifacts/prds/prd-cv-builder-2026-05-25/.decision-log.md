# Decision Log

## 2026-05-25 — PRD Creation

### D001: PRD Scope Level
- **Decision:** Comprehensive PRD with 21 FRs, 7 Success Metrics, 5 Cross-Cutting NFR sections. Fast path — drafted from extensive research and 7-agent party mode input.
- **Rationale:** Product is production-grade, investor-ready. PRD must be deep enough for UX design and architecture to start without re-litigating decisions.
- **Status:** Draft.

### D002: PRD Structure
- **Decision:** Used Essential Spine + Adapt-In Menu selections. Adapted template aggressively — added Monetization, Why Now, Constraints and Guardrails sections. Dropped Aesthetic and Tone (already in brief/design system research), Information Architecture (defer to UX design), Stakeholders (B2C — no approvals needed).
- **Rationale:** Sections added where the product carries concerns the template doesn't name. Sections dropped where they duplicate existing artifacts or don't serve B2C launch.
- **Status:** Confirmed.

### D003: UJ Count and Shape
- **Decision:** 3 User Journeys: UJ-1 (fresh grad first CV), UJ-2 (tech talent job-specific optimization), UJ-3 (viral share/referral).
- **Rationale:** UJ-1 covers the core acquisition and activation flow. UJ-2 covers the power-user retention flow. UJ-3 covers the growth loop. Three journeys is enough — additional personas (Dimas, Maya, Bu Dewi, Fajar) are V2+ and would dilute V1 focus.
- **Status:** Confirmed.

### D004: AI Interview — Structured Question Tree (Not Free-Form)
- **Decision:** The AI interview uses a structured question tree with dynamic follow-ups and output validation, not a fully free-form LLM conversation.
- **Rationale:** Predictable quality, debuggable, consistent extraction. Fully free-form LLM would produce inconsistent CV quality and make ATS pass rate guarantees impossible. This is a hybrid: structured state machine + LLM-powered natural language within each question.
- **Alternatives considered:** Fully free-form (rejected — quality risk). Fully scripted form (rejected — loses the "conversational" differentiator).
- **Status:** Confirmed.

### D005: PII Never Sent to LLM APIs
- **Decision:** Name, email, phone, address, photo, NIK/KTP are stripped before prompts and injected at PDF/DOCX rendering stage.
- **Rationale:** UU PDP compliance. AI provider data retention policies. Defense-in-depth — even if LLM API is compromised, PII is not exposed.
- **Status:** Confirmed — architecture must enforce this at the API gateway level.

### D006: DOCX Deferred to V2
- **Decision:** PDF-only in V1. DOCX added in V2.
- **Rationale:** Additional engineering surface (separate generation pipeline, template mapping, ATS compatibility testing for DOCX format). Open question logged: Indonesian HR preference for DOCX is real. User feedback may escalate.
- **Status:** Confirmed for V1. Marked as Open Question #4 for revisit.

### D007: 3 Templates for V1
- **Decision:** Professional, Modern, Minimal — three single-column ATS-safe templates.
- **Rationale:** All competitors start with 1-3 templates. Differentiation comes from ATS optimization depth, not template count. Template marketplace (V3) is the right time to expand.
- **Status:** Confirmed. Marked as Open Question #3 for beta validation.

### D008: V1 Pricing — Rp 49K Pro / Rp 75K Premium
- **Decision:** Pro Rp 49K/mo, Premium Rp 75K/mo. Annual discounts: 24% and 17%.
- **Rationale:** Research-backed (market research, competitor pricing, Indonesian WTP data). Below all Western competitors ($7-35/mo). Psychology: under Rp 50K threshold for Pro.
- **Status:** Draft — must validate with live A/B test within 2 weeks of launch. Open Question #5.

### D009: Business Model Risk Noted
- **Decision:** Victor's critique (unemployed Gen Z have limited WTP) is valid and logged. PRD includes monetization for B2C. B2B/gov revenue path acknowledged as Open Question #1.
- **Rationale:** V1 must prove B2C product-market fit first. B2B revenue (university, government, employer) is V2-V3 strategy. Building B2B before B2C PMF is premature.
- **Status:** Acknowledged — Open Question #1. Revisit at V2 gate.

### D010: Quality Gates
- **Decision:** 7 production readiness gates adopted from Murat's input: visual regression ≥99.5%, contract testing 100%, state machine coverage ≥90%, queue resilience 0 data loss, a11y 0 critical, performance budget verified, E2E smoke test 3 environments.
- **Rationale:** PRD for a product that handles career-critical documents must include testability requirements. Gates are non-negotiable before launch.
- **Status:** Confirmed.

### D011: PWA (Not Native) for V1-V2
- **Decision:** Mobile-first PWA with offline support. No native iOS/Android apps.
- **Rationale:** 90%+ Indonesian mobile traffic. PWA provides install-to-homescreen, offline, push notifications. Native apps add development cost and app store friction without clear V1 advantage. Revisit at V3 if PWA adoption or capability gaps warrant.
- **Status:** Confirmed.
