# Decision Log

## 2026-05-24 — Initial Brief Creation

### D001: Product Name
- **Decision:** "Lolos" (Indonesian: "pass through" / "get in")
- **Alternatives considered:** None explicitly — emerged from brainstorming
- **Rationale:** Culturally resonant, SEO-distinctive, works for ID and global markets
- **Status:** Draft — needs validation with target users

### D002: Multi-User (Not Multi-Tenant)
- **Decision:** B2C multi-user architecture — no tenant isolation
- **Rationale:** Product serves individual job seekers, not organizations. Simpler architecture, faster development, lower cost.
- **Status:** Confirmed

### D003: Multi-Template System
- **Decision:** Pluggable template system — users pick template after AI interview, switch anytime without data loss
- **Rationale:** Different job types need different CV formats (corporate vs. startup vs. academic). Multi-template is a competitive differentiator.
- **Status:** Confirmed

### D004: AI Credit System (Not Unlimited AI)
- **Decision:** Hybrid credit + subscription model — 50/300/600 monthly credits per tier
- **Rationale:** Prevents API cost abuse at scale. Creates natural upgrade triggers. Competitor analysis shows unlimited AI models risk margin erosion.
- **Status:** Confirmed — monitor cost per free user, adjust credit allocation if conversion suffers

### D005: Bahasa Indonesia First, English Second
- **Decision:** Launch with Bahasa Indonesia only. Add English in V2.
- **Rationale:** Primary market is Indonesia. Zero competitors serve Bahasa Indonesia. English is a secondary expansion, not a launch requirement.
- **Status:** Confirmed

### D006: Mobile-First (Not Responsive Desktop)
- **Decision:** Design for mobile first, desktop as secondary. PWA with offline support.
- **Rationale:** 90%+ Indonesian internet traffic is mobile. Every competitor is desktop-first. Mobile-first is a differentiator that is expensive to retrofit.
- **Status:** Confirmed

### D007: Technology Stack
- **Decision:** Next.js 14+ App Router, NestJS/TypeScript, PostgreSQL 16 + pgvector, Redis, BullMQ, TipTap editor
- **Rationale:** Validated through 9-pillar technical research. End-to-end TypeScript type safety via tRPC. pgvector sufficient for current scale (no dedicated vector DB).
- **Status:** Confirmed

### D008: Cloud Provider
- **Decision:** Alibaba Cloud Jakarta (primary, cost-optimized) or AWS Jakarta (compliance-optimized)
- **Rationale:** Both satisfy UU PDP data residency. Alibaba Cloud 10-25% cheaper. AWS has 4 SNI certifications (unique advantage).
- **Status:** Deferred — decide based on final compliance requirements and cost modeling

### D009: Payment Methods
- **Decision:** GoPay + QRIS + Virtual Account via Xendit. Stripe for international users in V2.
- **Rationale:** Only 2% credit card penetration in Indonesia. 78% use e-wallets. Xendit has best recurring billing API for local methods.
- **Status:** Confirmed

### D010: Freemium Model
- **Decision:** Generous free tier (1 complete CV, 50 AI credits, PDF with subtle watermark). Pro: IDR 49K/mo. Premium: IDR 75K/mo.
- **Rationale:** "Pay to download" model (Zety, Resume Genius) generates negative brand sentiment. Free tier builds trust and creates upgrade triggers (ATS score reveal, second CV, DOCX export).
- **Status:** Confirmed — monitor conversion rate, adjust if below 5% target

### D011: DOCX in V2 (Not V1)
- **Decision:** Ship PDF-only in V1. Add DOCX in V2.
- **Rationale:** PDF is universal for ATS applications. DOCX adds engineering complexity (separate generation pipeline, template system, ATS compatibility testing). Indonesian HR demand for DOCX is real but can be addressed in V2.
- **Status:** Confirmed — user feedback may escalate if HR departments reject PDF-only

### D012: Enterprise/University in V3
- **Decision:** No enterprise or university features in V1/V2. B2C only.
- **Rationale:** B2B sales cycle adds complexity. University partnerships require relationship building. Focus on consumer product-market fit first.
- **Status:** Confirmed — re-evaluate at V2 gate if university inbound demand is significant
